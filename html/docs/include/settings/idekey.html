Controls which IDE Key Xdebug should pass on to the debugging client or proxy. The IDE Key is only important for use with the [FEAT:dbgpProxy], although some IDEs are incorrectly picky as to what its value is.

The default is based on the DBGP_IDEKEY environment setting. If it is not present, the default falls back to an empty string.

If this setting is set to a non-empty string, it selects its value over DBGP_IDEKEY environment variable as default value.

The internal IDE Key also gets updated through debugging session management and overrides the value of this setting as is explained in the [FEAT:step_debug#activate_debugger] documentation.

This setting can additionally be configured through the XDEBUG_CONFIG environment variable.
